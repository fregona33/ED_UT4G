La ACTIVIDAD UNIDAD 4 describe una práctica grupal centrada en la optimización, refactorización y documentación de código en un entorno colaborativo. La actividad, planteada como una misión denominada “Operación Hotel Espagueti”, sitúa al alumnado en un contexto profesional donde deben enfrentarse a un código heredado con alta deuda técnica. Los objetivos de aprendizaje se enfocan en el trabajo en equipo con repositorios remotos, el uso de Git y ramas, la escritura de pruebas unitarias, la aplicación de refactorizaciones avanzadas y la generación de documentación XML profesional en C#, todo ello reforzando buenas prácticas de desarrollo de software.

La práctica se estructura en cinco fases progresivas: preparación del entorno colaborativo, creación de pruebas unitarias, refactorización en paralelo para provocar conflictos de fusión, resolución guiada de dichos conflictos y autodocumentación del código final. Cada alumno asume responsabilidades diferenciadas para fomentar la coordinación y la integración de cambios, finalizando con una entrega conjunta que consiste en un informe profesional en PDF, con descripción detallada del proceso seguido, problemas encontrados, enlace al repositorio y una conclusión con propuestas de mejora. La entrega está limitada a un único miembro del grupo y debe cumplir estrictamente los requisitos de formato y contenido establecidos.
